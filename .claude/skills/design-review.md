# スライドデザインレビュー

作成済みのMarpスライドを分析し、デザイン改善を提案します。

## レビュー観点

### 1. コンポーネント選択
- 箇条書きが多い → kg-two-col で2カラム化を検討
- ラベル+値のペア → kg-spec の使用を提案
- 重要ポイント → kg-callout で強調
- 複数の関連項目 → kg-card でグループ化
- 比較データ → kg-table で表形式に

### 2. レイアウト
- 情報量が多すぎないか（1スライド1メッセージ）
- 余白は適切か
- 視線の流れは自然か（左上→右下）

### 3. スライドクラスの使い分け
- カバーに title クラスを使用しているか
- セクション区切りに top クラスを使用しているか
- 通常コンテンツで不要なclass指定がないか

### 4. 視覚的階層
- h1, h2, h3 の使い分けは適切か
- 強調すべき箇所が埋もれていないか
- kg-badge で重要度を示せないか

### 5. 一貫性
- コンポーネントの使い方が統一されているか
- フッターの著作権表記があるか

## 指示

1. 対象のマークダウンファイルを読み込む
2. 各スライドを上記の観点でレビュー
3. 問題点と改善案を具体的に提示
4. 改善後のコード例を提供
5. 優先度（高/中/低）を付けて整理
